## HRMS-App-Automation Project

### Project Overview

Developed and maintained an end-to-end test automation framework for an HRMS (Human Resource Management System) web application using Playwright. The project focused on automating critical HR workflows such as employee management, payroll processing, leave management, authentication, and role-based access functionalities to improve software quality and reduce manual testing effort.

### Technologies Used

* Playwright
* JavaScript / TypeScript
* Node.js
* HTML Reporting
* Git & GitHub
* Jenkins / CI-CD Integration
* Azure DevOps
* REST API Validation
* YAML Configuration
* Cross-Browser Testing

### Roles & Responsibilities

* Designed and implemented scalable automation test frameworks using Playwright.
* Automated end-to-end functional test cases for HRMS modules including:

  * Employee Onboarding
  * Leave Management
  * Payroll
  * Attendance Tracking
  * User Authentication & Authorization
  * Role-Based Access Control
* Developed reusable page object models (POM) to improve framework maintainability and code reusability.
* Performed cross-browser testing on Chromium, Firefox, and WebKit browsers.
* Implemented data-driven testing using external test data files.
* Integrated automated test execution into CI/CD pipelines using Jenkins and Azure DevOps.
* Executed smoke, regression, and sanity test suites for multiple releases.
* Logged and tracked defects using Agile methodologies and collaborated with developers for issue resolution.
* Performed API response validation alongside UI testing for end-to-end workflow verification.

### Reporting & Test Execution

Implemented advanced reporting and logging mechanisms to provide detailed test execution insights and improve debugging efficiency.

#### Reporting Features

* Generated detailed HTML reports after every test execution.
* Integrated Playwright Trace Viewer for debugging failed test scenarios.
* Captured automatic screenshots for failed test cases.
* Recorded video execution of test runs for easier defect analysis.
* Maintained execution logs for troubleshooting and auditing purposes.
* Published test execution summaries in CI/CD pipelines.
* Configured retry mechanisms and failure analysis reporting.

#### Report Benefits

* Improved visibility into automation test results.
* Reduced debugging time through screenshots, logs, and video evidence.
* Enabled faster identification of failed workflows and UI issues.
* Provided stakeholders with clear execution summaries and pass/fail metrics.

### Project Outcome

* Reduced manual regression testing effort significantly.
* Improved application stability and release confidence.
* Accelerated testing cycles through automation and CI/CD integration.
* Enhanced test coverage for critical HRMS business workflows.
